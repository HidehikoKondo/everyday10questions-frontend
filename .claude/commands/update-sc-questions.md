# 情報処理安全確保支援士問題更新スキル

情報処理安全確保支援士試験の2カテゴリ（午前Ⅰ・午前Ⅱ）の問題を新たに10問ずつ生成し、JSONファイルを更新する。

## 手順

### 1. 参照資料の確認（任意）
IPA公式サイトの出題範囲・過去問が利用可能であれば、WebFetchで取得して問題作成の参考にすること。
取得できない場合はスキップして問題作成を続行すること。

### 2. 問題の生成ルール

**共通ルール：**
- 各カテゴリ10問ずつ作成する
- 過去問を参考にした**予想問題**として作成する（実際の過去問をそのまま転用しない）
- 問題文・選択肢・解説はすべて日本語で記述する
- ヒントは150文字程度、解説は200文字程度とする
- `answer` は `"choice1"` 〜 `"choice4"` のいずれかとし、正解が特定の選択肢番号に偏らないようにする
- 高度試験レベルの**難易度**で設定する（深い専門知識・実践的判断を問う問題を含める）

**JSONフォーマット（厳守）：**
```json
{
  "questions": [
    {
      "question": "問題文",
      "answer": "choice1",
      "hint": "ヒントのテキスト（150文字程度）",
      "explanation": "解説のテキスト（200文字程度）",
      "choice1": "選択肢1",
      "choice2": "選択肢2",
      "choice3": "選択肢3",
      "choice4": "選択肢4",
      "scope": "カテゴリ名（例：午前Ⅱ）",
      "text": "出題範囲（例：暗号・認証技術）"
    }
  ]
}
```

### 3. 午前Ⅰ（gozen1_questions.json）

高度試験共通の出題範囲。次のカテゴリからまんべんなく各1問以上含めること：
- テクノロジ（基礎理論・アルゴリズム・コンピュータ科学・データベース・ネットワーク）
- セキュリティ（暗号・認証・攻撃手法・セキュリティ管理）
- マネジメント（プロジェクトマネジメント・サービスマネジメント・システム監査）
- ストラテジ（経営戦略・情報戦略・法務・IT戦略）

`scope` は `"午前Ⅰ"` の表記に統一する。
`text` は上記カテゴリ名を簡潔に記述する。

生成後、以下のファイルを**完全に置き換える**：
`questions/sc/gozen1_questions.json`

### 4. 午前Ⅱ（gozen2_questions.json）

情報セキュリティ専門の出題範囲。次のカテゴリからまんべんなく各1問以上含めること：
- 暗号・認証技術（共通鍵・公開鍵暗号、ハッシュ関数、デジタル署名、PKI、TLS）
- ネットワークセキュリティ（ファイアウォール、IDS/IPS、WAF、VPN、DNSセキュリティ）
- システムセキュリティ（アクセス制御、脆弱性管理、セキュアコーディング、ログ管理）
- 脅威・攻撃手法（マルウェア、APT、Webアプリ攻撃、フィッシング、標的型攻撃）
- セキュリティ管理・監査（ISMS、リスクマネジメント、BCP、脆弱性診断）
- 法的規制・ガイドライン（個人情報保護法、不正アクセス禁止法、サイバーセキュリティ基本法、NIST・ISOガイドライン）

`scope` は `"午前Ⅱ"` の表記に統一する。
`text` は上記カテゴリ名を簡潔に記述する。

生成後、以下のファイルを**完全に置き換える**：
`questions/sc/gozen2_questions.json`

### 5. 完了報告

2ファイルの更新が完了したら、以下を報告する：
- 更新したファイル名と問題数
- 各カテゴリで含めた出題範囲の一覧
- 更新日時（today's date）
