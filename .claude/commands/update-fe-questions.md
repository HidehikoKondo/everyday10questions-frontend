# 基本情報技術者試験問題更新スキル

基本情報技術者試験の3カテゴリ（テクノロジ系・マネジメント系・ストラテジ系）の問題を新たに10問ずつ生成し、JSONファイルを更新する。

## 手順

### 1. 参照資料の確認（任意）
IPA公式サイトの出題範囲・過去問が利用可能であれば、WebFetchで取得して問題作成の参考にすること。
取得できない場合はスキップして問題作成を続行すること。

### 2. 問題の生成ルール

**共通ルール：**
- 各カテゴリ10問ずつ作成する
- 過去問を参考にした**予想問題**として作成する（実際の過去問をそのまま転用しない）
- 問題文・選択肢・解説はすべて日本語で記述する
- ヒントは150文字程度、解説は200文字程度とする
- `answer` は `"choice1"` 〜 `"choice4"` のいずれかとし、正解が特定の選択肢番号に偏らないようにする
- ITパスポートより**難易度を高く**設定する（技術的な詳細知識を問う問題を含める）

**JSONフォーマット（厳守）：**
```json
{
  "questions": [
    {
      "question": "問題文",
      "answer": "choice1",
      "hint": "ヒントのテキスト（150文字程度）",
      "explanation": "解説のテキスト（200文字程度）",
      "choice1": "選択肢1",
      "choice2": "選択肢2",
      "choice3": "選択肢3",
      "choice4": "選択肢4",
      "scope": "カテゴリ名（例：テクノロジ系）",
      "text": "出題範囲（例：データベース）"
    }
  ]
}
```

### 3. テクノロジ系（technology_questions.json）

次のカテゴリからまんべんなく各1問以上含めること：
- 基礎理論（数学・論理演算・情報理論・オートマトン・計算量）
- アルゴリズムとプログラミング（アルゴリズム、データ構造、プログラミング言語）
- コンピュータ構成要素（CPU、メモリ、I/Oデバイス、処理方式）
- システム構成要素（システム性能、信頼性、RAID、仮想化）
- ソフトウェア（OS、ミドルウェア、ファイルシステム）
- データベース（関係データベース、SQL、トランザクション、正規化）
- ネットワーク（TCP/IP、プロトコル、LANとWAN、ルーティング）
- セキュリティ（暗号、認証、攻撃手法、セキュリティ対策）

`scope` は `"テクノロジ系"` の表記に統一する。
`text` は上記カテゴリ名を簡潔に記述する。

生成後、以下のファイルを**完全に置き換える**：
`questions/fe/technology_questions.json`

### 4. マネジメント系（management_questions.json）

次のカテゴリからまんべんなく各1問以上含めること：
- システム開発技術（開発プロセス、要件定義、設計、テスト手法、品質管理）
- ソフトウェア開発管理技術（開発モデル、CMMI、構成管理、レビュー）
- プロジェクトマネジメント（PMBOK、WBS、スケジュール管理、コスト管理）
- サービスマネジメント（ITIL、SLA、インシデント管理、変更管理）
- システム監査（監査の目的・手順、内部統制、COBIT）

`scope` は `"マネジメント系"` の表記に統一する。
`text` は上記カテゴリ名を簡潔に記述する。

生成後、以下のファイルを**完全に置き換える**：
`questions/fe/management_questions.json`

### 5. ストラテジ系（strategy_questions.json）

次のカテゴリからまんべんなく各1問以上含めること：
- 経営戦略マネジメント（経営戦略、マーケティング、経営管理手法）
- 技術戦略マネジメント（技術開発戦略、MOT、知的財産）
- ビジネスインダストリ（業種・業態、電子商取引、IoT・AI活用）
- システム戦略（情報システム戦略、業務改革、SoA）
- システム企画（システム化計画、要件定義、調達・契約）
- 法務（知的財産権、セキュリティ関連法規、労働関連法規、その他関連法規）
- 企業活動（財務・会計、組織論、OR・IE手法）

`scope` は `"ストラテジ系"` の表記に統一する。
`text` は上記カテゴリ名を簡潔に記述する。

生成後、以下のファイルを**完全に置き換える**：
`questions/fe/strategy_questions.json`

### 6. 完了報告

3ファイルの更新が完了したら、以下を報告する：
- 更新したファイル名と問題数
- 各カテゴリで含めた出題範囲の一覧
- 更新日時（today's date）
