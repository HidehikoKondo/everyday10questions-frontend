# 基本情報技術者試験問題更新スキル

基本情報技術者試験の科目A（テクノロジ系・マネジメント系・ストラテジ系）と科目B（アルゴリズム・情報セキュリティ）の問題を新たに生成し、JSONファイルを更新する。

## 手順

### 1. 参照資料の確認（任意）
IPA公式サイトの出題範囲・過去問が利用可能であれば、WebFetchで取得して問題作成の参考にすること。
取得できない場合はスキップして問題作成を続行すること。

### 2. 問題の生成ルール

**共通ルール：**
- 各カテゴリ10問ずつ作成する
- 過去問を参考にした**予想問題**として作成する（実際の過去問をそのまま転用しない）
- 問題文・選択肢・解説はすべて日本語で記述する
- ヒントは150文字程度、解説は200文字程度とする
- 解説で選択肢を表現するときはchoice1ではなく[1]と表現してください。
- `answer` は `"choice1"` 〜 `"choice4"` のいずれかとし、正解が特定の選択肢番号に偏らないようにする
- `keyword` は問題文中に登場するより大きいテーマの言葉を優先して選定する（例：「統制活動」より「内部統制」、「スプリントレビュー」より「アジャイル開発」）
- ITパスポートより**難易度を高く**設定する（技術的な詳細知識を問う問題を含める）

**JSONフォーマット（厳守）：**
```json
{
  "questions": [
    {
      "question": "問題文",
      "answer": "choice1",
      "hint": "ヒントのテキスト（150文字程度）",
      "explanation": "解説のテキスト（200文字程度）",
      "choice1": "選択肢1",
      "choice2": "選択肢2",
      "choice3": "選択肢3",
      "choice4": "選択肢4",
      "category": "カテゴリ名（例：テクノロジ系）",
      "topic": "出題範囲（例：データベース）",
      "keyword": "第1正規形"
    }
  ]
}
```

### 3. テクノロジ系（technology_questions.json）

次のカテゴリからまんべんなく各1問以上含めること：
- 基礎理論（数学・論理演算・情報理論・オートマトン・計算量）
- アルゴリズムとプログラミング（アルゴリズム、データ構造、プログラミング言語）
- コンピュータ構成要素（CPU、メモリ、I/Oデバイス、処理方式）
- システム構成要素（システム性能、信頼性、RAID、仮想化）
- ソフトウェア（OS、ミドルウェア、ファイルシステム）
- データベース（関係データベース、SQL、トランザクション、正規化）
- ネットワーク（TCP/IP、プロトコル、LANとWAN、ルーティング）
- セキュリティ（暗号、認証、攻撃手法、セキュリティ対策）

`category` は `"テクノロジ系"` の表記に統一する。
`topic` は上記カテゴリ名を簡潔に記述する。

生成後、以下のファイルを**完全に置き換える**：
`questions/fe/technology_questions.json`

### 4. マネジメント系（management_questions.json）

次のカテゴリからまんべんなく各1問以上含めること：
- システム開発技術（開発プロセス、要件定義、設計、テスト手法、品質管理）
- ソフトウェア開発管理技術（開発モデル、CMMI、構成管理、レビュー）
- プロジェクトマネジメント（PMBOK、WBS、スケジュール管理、コスト管理）
- サービスマネジメント（ITIL、SLA、インシデント管理、変更管理）
- システム監査（監査の目的・手順、内部統制、COBIT）

`category` は `"マネジメント系"` の表記に統一する。
`topic` は上記カテゴリ名を簡潔に記述する。

生成後、以下のファイルを**完全に置き換える**：
`questions/fe/management_questions.json`

### 5. ストラテジ系（strategy_questions.json）

次のカテゴリからまんべんなく各1問以上含めること：
- 経営戦略マネジメント（経営戦略、マーケティング、経営管理手法）
- 技術戦略マネジメント（技術開発戦略、MOT、知的財産）
- ビジネスインダストリ（業種・業態、電子商取引、IoT・AI活用）
- システム戦略（情報システム戦略、業務改革、SoA）
- システム企画（システム化計画、要件定義、調達・契約）
- 法務（知的財産権、セキュリティ関連法規、労働関連法規、その他関連法規）
- 企業活動（財務・会計、組織論、OR・IE手法）

`category` は `"ストラテジ系"` の表記に統一する。
`topic` は上記カテゴリ名を簡潔に記述する。

生成後、以下のファイルを**完全に置き換える**：
`questions/fe/strategy_questions.json`

### 6. 科目B（scienceb_questions.json）

実際の試験に即した出題形式で作成する：
- **アルゴリズム問題（8問）**: 擬似言語でプログラムを提示し、実行結果・変数の値・戻り値などを問う
- **情報セキュリティ問題（2問）**: インシデント対応・セキュアコーディング・セキュリティ設計などの実践的判断を問う

**擬似言語の記法ルール：**
- 変数宣言: `整数型: x`、`文字列型: s`、`整数型の配列: a`
- 代入: `x ← 5`
- 条件分岐: `if (条件) ... elseif (条件) ... else ... endif`
- for ループ: `for (i を 1 から n まで 1 ずつ増やす) ... endfor`
- while ループ: `while (条件) ... endwhile`
- 関数定義: `○ 戻り型 関数名(引数型: 引数名)`
- 配列の添字は1から始まる
- 剰余: `mod`、整数除算の商: `÷`（小数点以下切り捨て）

**問題作成の注意点：**
- 問題文に擬似言語コードを記載し、その実行結果を選択肢から選ぶ形式にする
- 手でトレースできる程度の規模（ループ回数10回以内、再帰深さ5以内）にする
- 選択肢は4つの具体的な値・文字列・選択肢記号（A〜D）のいずれかとする
- `question` フィールドに擬似言語コードも含めて記述する（改行は `\n` で表現）
- コードブロックの前後には空行（`\n\n`）を入れて、説明文とコードを視覚的に分離する
- インデントには全角スペース（　）を使用する（半角スペースはHTMLで圧縮されるため）
- `keyword` はアルゴリズム種別（例：整列アルゴリズム、再帰アルゴリズム、データ構造）またはセキュリティ技術名を記載する

`category` は `"科目B"` の表記に統一する。
`topic` はアルゴリズム問題は `"アルゴリズム"`、セキュリティ問題は `"情報セキュリティ"` とする。

生成後、以下のファイルを**完全に置き換える**：
`questions/fe/scienceb_questions.json`
