# 情報セキュリティマネジメント問題更新スキル

情報セキュリティマネジメント試験の科目A・科目Bの問題を新たに10問ずつ生成し、JSONファイルを更新する。

## 手順

### 1. 参照資料の確認（任意）
IPA公式サイトの出題範囲・過去問が利用可能であれば、WebFetchで取得して問題作成の参考にすること。
取得できない場合はスキップして問題作成を続行すること。

### 2. 問題の生成ルール

**共通ルール：**
- 各ファイル10問ずつ作成する
- 過去問を参考にした**予想問題**として作成する（実際の過去問をそのまま転用しない）
- 問題文・選択肢・解説はすべて日本語で記述する
- ヒントは150文字程度、解説は200文字程度とする
- 解説で選択肢を表現するときはchoice1ではなく[1]と表現してください。
- `answer` は `"choice1"` 〜 `"choice4"` のいずれかとし、正解が特定の選択肢番号に偏らないようにする
- `keyword` は問題文中に登場するより大きいテーマの言葉を優先して選定する（例：「統制活動」より「内部統制」、「スプリントレビュー」より「アジャイル開発」）

**JSONフォーマット（厳守）：**
```json
{
  "questions": [
    {
      "question": "問題文",
      "answer": "choice1",
      "hint": "ヒントのテキスト（150文字程度）",
      "explanation": "解説のテキスト（200文字程度）",
      "choice1": "選択肢1",
      "choice2": "選択肢2",
      "choice3": "選択肢3",
      "choice4": "選択肢4",
      "category": "カテゴリ名",
      "topic": "出題範囲（例：暗号技術）",
      "keyword": "公開鍵暗号方式"
    }
  ]
}
```

### 3. 科目A（sg_questions.json）

情報セキュリティ全般の知識を問う4択問題。次のカテゴリからまんべんなく各1問以上含めること：
- 情報セキュリティ管理（ISMS、リスクマネジメント、セキュリティポリシー）
- 情報セキュリティ対策（人的・技術的・物理的対策、インシデント対応）
- 暗号技術（共通鍵・公開鍵暗号、ハッシュ関数、デジタル署名、PKI）
- 認証技術（認証方式、多要素認証、バイオメトリクス）
- ネットワークセキュリティ（ファイアウォール、IDS/IPS、VPN、TLS）
- 脅威・脆弱性（マルウェア、攻撃手法、ソーシャルエンジニアリング）
- 法的規制・ガイドライン（個人情報保護法、不正アクセス禁止法、サイバーセキュリティ基本法）

`category` は `"科目A"` の表記に統一する。
`topic` は上記カテゴリ名を簡潔に記述する（例：`"暗号技術"`、`"法的規制"`）。

生成後、以下のファイルを**完全に置き換える**：
`questions/sg/sg_questions.json`

### 4. 科目B（scienceb_questions.json）

情報セキュリティ管理の実践を問う**シナリオ（事例）形式**の問題。
実際の試験と同様に、企業や組織の状況を示すシナリオ文を提示し、その状況に対する適切な判断・対応を問う形式で作成する。

**問題形式のルール：**
- `question` フィールドにシナリオ文（企業・組織の状況説明）と質問を含める
- シナリオは現実的な企業の情報セキュリティ場面を設定する（例：インシデント発生、リスク評価、ポリシー整備など）
- 単純な知識ではなく、シナリオに基づく**実践的な判断・分析**を問う
- 選択肢はすべて「それらしい」内容にして、知識だけでは判断しにくい実践問題にする

**出題テーマ（各1問以上含めること）：**
- インシデント対応（検知・初動・報告・復旧手順の判断）
- リスクアセスメント（リスク特定・分析・評価・対応方針の選択）
- セキュリティポリシーの運用（ポリシー違反への対応、例外申請処理など）
- 委託先管理（クラウド・外部委託のセキュリティ管理）
- 情報資産管理（分類・取り扱い手順・廃棄管理）
- 教育・訓練（セキュリティ教育の計画・実施・効果測定）
- 監査・点検（内部監査の実施・指摘事項への対応）
- 事業継続（BCP・BCM・DRの観点からのセキュリティ対応）

`category` は `"科目B"` の表記に統一する。
`topic` は上記テーマ名を簡潔に記述する（例：`"インシデント対応"`、`"リスクアセスメント"`）。

生成後、以下のファイルを**完全に置き換える**：
`questions/sg/scienceb_questions.json`

### 5. 完了報告

2ファイルの更新が完了したら、以下を報告する：
- 更新したファイル名と問題数
- 各ファイルで含めた出題範囲の一覧
- 更新日時（today's date）
