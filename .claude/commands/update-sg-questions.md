# 情報セキュリティマネジメント問題更新スキル

情報セキュリティマネジメント試験の問題を新たに10問生成し、JSONファイルを更新する。

## 手順

### 1. 参照資料の確認（任意）
IPA公式サイトの出題範囲・過去問が利用可能であれば、WebFetchで取得して問題作成の参考にすること。
取得できない場合はスキップして問題作成を続行すること。

### 2. 問題の生成ルール

**共通ルール：**
- 10問作成する
- 過去問を参考にした**予想問題**として作成する（実際の過去問をそのまま転用しない）
- 問題文・選択肢・解説はすべて日本語で記述する
- ヒントは150文字程度、解説は200文字程度とする
- `answer` は `"choice1"` 〜 `"choice4"` のいずれかとし、正解が特定の選択肢番号に偏らないようにする

**JSONフォーマット（厳守）：**
```json
{
  "questions": [
    {
      "question": "問題文",
      "answer": "choice1",
      "hint": "ヒントのテキスト（150文字程度）",
      "explanation": "解説のテキスト（200文字程度）",
      "choice1": "選択肢1",
      "choice2": "選択肢2",
      "choice3": "選択肢3",
      "choice4": "選択肢4",
      "scope": "情報セキュリティマネジメント",
      "text": "出題範囲（例：暗号技術）"
    }
  ]
}
```

### 3. 情報セキュリティマネジメント（sg_questions.json）

次のカテゴリからまんべんなく各1問以上含めること：
- 情報セキュリティ管理（ISMS、リスクマネジメント、セキュリティポリシー）
- 情報セキュリティ対策（人的・技術的・物理的対策、インシデント対応）
- 暗号技術（共通鍵・公開鍵暗号、ハッシュ関数、デジタル署名、PKI）
- 認証技術（認証方式、多要素認証、バイオメトリクス）
- ネットワークセキュリティ（ファイアウォール、IDS/IPS、VPN、TLS）
- 脅威・脆弱性（マルウェア、攻撃手法、ソーシャルエンジニアリング）
- 法的規制・ガイドライン（個人情報保護法、不正アクセス禁止法、サイバーセキュリティ基本法）

`scope` は `"情報セキュリティマネジメント"` の表記に統一する。
`text` は上記カテゴリ名を簡潔に記述する（例：`"暗号技術"`、`"法的規制"`）。

生成後、以下のファイルを**完全に置き換える**：
`questions/sg/sg_questions.json`

### 4. 完了報告

ファイルの更新が完了したら、以下を報告する：
- 更新したファイル名と問題数
- 含めた出題範囲の一覧
- 更新日時（today's date）
